서비스명
LoveType

서비스 한 줄 정의
사용자의 성향과 상징 데이터를 바탕으로
오늘의 흐름과 감정을 하나의 이야기로 읽어주는
서사형 타로 리딩 앱

1. 서비스 핵심 방향

LoveType은 전통적인 운세 앱보다
개인 맞춤형 이야기와 감정 몰입에 초점을 둔다.

핵심 방향
- 운세보다 이야기
- 예언보다 흐름
- 자극보다 감정 몰입
- 공포보다 따뜻한 통찰
- 소비형 점술보다 내 공간 같은 앱

핵심 인상
- 이 앱은 내 정보를 기억한다
- 이 앱은 나를 상징하는 카드를 가지고 있다
- 이 앱은 내 이야기를 쌓아간다

2. MVP 목표

MVP 목표는
사용자가 프로필 입력 후
자신의 소울카드를 확인하고
오늘의 이야기 또는 연애 이야기 중 하나를 선택해
3장의 카드를 뽑고
개인화된 이야기를 읽고
저장/공유까지 경험하는 것이다.

MVP에서 가장 중요한 경험
1. 소울카드 첫 등장
2. 카드 3장 선택 연출
3. 이야기 결과의 품질
4. 저장/공유를 통한 애착 형성

3. MVP 카테고리 범위

MVP 포함
- 오늘의 이야기
- 연애 이야기

확장 예정
- 일의 이야기
- 이동의 이야기
- 배움의 이야기

카테고리 역할

오늘의 이야기
- 서비스 신뢰 형성용 기본 카테고리
- 하루 1회 진입의 대표 카테고리
- 부담 없이 앱 품질을 체감하게 하는 입구
- 오늘의 감정, 흐름, 선택, 관계, 타이밍 중심

연애 이야기
- 몰입과 개인화 체감을 강화하는 핵심 카테고리
- 저장/공유/유료 전환 가능성이 높은 카테고리
- 썸, 연락, 재회, 고백, 상대마음 중심

전략
오늘의 이야기로 앱이 탄탄하다는 인상을 만들고
연애 이야기로 더 깊이 들어가게 만든다.

5. 소울카드 정의

소울카드는 사용자를 상징하는 대표 카드다.
단순 계산 결과가 아니라
앱 안에서 사용자의 정체성과 애착을 형성하는 핵심 장치다.

소울카드 역할
- 개인화의 상징
- 앱 안의 아바타
- 내 맞춤 앱이라는 느낌 강화
- 모든 이야기의 출발점

핵심 연출
- 회원정보 입력 완료 후 소울카드 등장
- 메시지:
  "이것은 당신을 상징하는 카드입니다."

표현 방식
- 카드 일러스트
- 카드명
- 짧은 정체성 문구
- 닉네임과 함께 표시

예시
[별 카드]
인블루
희망을 오래 붙드는 사람

UI 방향
- 홈 상단에 소울카드 프로필 고정
- 이야기 선택 화면에도 미니 카드 배지 노출
- 결과 화면에도 소울카드 연결 표시
- 히스토리 전체 톤에도 소울카드 스타일 반영

핵심 목표
사용자가 LoveType을 단순한 앱이 아니라
내 이야기가 쌓이는 개인 공간처럼 느끼게 한다.

- 결과는 읽기 쉽고 저장하고 싶은 길이로 제공
- 저장/공유로 애착과 바이럴 유도

8. 리딩 생성 구조

핵심 원칙
LLM 호출은 1회만 사용한다.
해석의 판단은 앱과 JSON 데이터 조합기가 맡고
LLM은 마지막 문장화만 담당한다.

사전 준비 JSON
- theme_keywords.json
- gender_keywords.json
- tarot_cards.json
- soul_cards.json
- bloodtype_keywords.json
- mbti_keywords.json
- focus_keywords.json

MVP 우선 대상
- today
- love

리딩 조합 요소
- theme
- gender
- mbti
- blood_type
- soul_card
- focus_keyword
- tarot_card 3장
- 각 카드 정/역방향
- character keywords
- theme frame
- tone guide

최종 payload를 만든 뒤
LLM에 1회 전달하여 결과를 생성한다.

LLM 역할
- 한 줄 총평 작성
- 현재 흐름 작성
- 조심할 포인트 작성
- 기대해볼 포인트 작성
- 공유용 짧은 문장 작성

결과 출력 형식
1. 한 줄 총평
2. 현재 흐름 2~3문장
3. 조심할 포인트 1문장
4. 기대해볼 포인트 1문장
5. 공유용 짧은 문장 1개

품질 원칙
- 카드 코어 의미는 흔들지 않는다
- 역방향도 과도하게 부정적으로 쓰지 않는다
- 조심, 약한 흐름, 점검 필요 정도로 표현한다
- 무섭게 말하지 않는다
- theme와 focus를 자연스럽게 녹인다

9. 결과 다양성 원칙

핵심 목표
1000번을 뽑아도 완전히 같은 결과가 반복되지 않게 한다.

방식
- 카드 의미는 고정
- 해석 관점은 다양화
- 문장 표현은 매번 새롭게
- 최근 사용 관점과 표현 중복 회피
- theme / focus / mbti / blood / soul_card 조합으로 차별화

원칙
무작위가 아니라
일관된 의미 안에서 관점과 언어를 달리하는 구조를 지향한다.

11. 카드 덱 구조

MVP 기본 무료 덱
- 일반 덱 1종
- 저시력용 고대비 덱 1종

MVP 프리미엄 덱
- 감성 테마 덱 1종
- 후보:
  - 웹툰 로맨스 덱
  - 서정 판타지 덱

제공 방식
- 단품 구매 2,000원
- 구독자 포함

저시력용 덱 방향
- 검정 배경
- 청록/노랑 계열 강조
- 큰 심볼
- 높은 대비
- 작은 화면에서도 식별 쉬움

카드 덱 전략
무료 사용자도 충분히 만족 가능한 기본 덱 2종을 제공하고
프리미엄 덱은 취향과 감성 확장 상품으로 설계한다.

13. 이용 제한 정책

무료
- 하루 1회

유료
- 같은 카테고리는 1시간 이내 재조회 불가

정책 표현
이 제한은 단순 차단이 아니라
타로의 흐름을 존중하는 규칙처럼 안내한다.

예시 방향
- 같은 흐름은 잠시 가라앉힌 뒤 다시 읽는 편이 좋습니다
- 지금의 카드는 아직 이야기를 이어가는 중입니다

14. 환불 정책 방향

핵심 원칙
- 기준은 노멀하게
- 적용은 관대하게

정책 철학
일반적인 디지털 콘텐츠 서비스 수준의 환불 기준을 따르되
초기 서비스 운영 단계에서는
사용자 불편을 줄이기 위해 보다 관대한 환불 운영을 한다.

내부 운영 방향
- 미사용 건은 거의 즉시 환불
- 사용 직후 요청도 특별한 문제 없으면 관대 승인
- 중복 결제 / 오작동 / 오인 결제는 우선 환불
- 반복적 악용 정황만 예외 검토

의도
꼼꼼한 이용자에게 과금 불안을 줄이고
초기 서비스 신뢰를 높인다.

16. 백엔드 방향

핵심 방향
MVP부터 백엔드를 붙인다.
다만 무겁게 만들지 않고
저장, 제한, 검증, 중계 역할만 맡기는 얇은 백엔드로 간다.

MVP 백엔드 역할
1. 사용자 프로필 저장
2. 소울카드 계산 결과 보관
3. 리딩 요청 중계
4. 사용 제한 관리
5. 히스토리 저장
6. 백업/복구
7. 결제 상태 확인(확장)

이유
- 로컬 구조를 나중에 서버로 다시 옮기는 이중 작업 방지
- 데이터 구조를 처음부터 정식 서비스 기준으로 고정
- 무료/유료 제한 및 백업 구조를 처음부터 일관되게 설계 가능
