# LoveType Tarot PRD

작성일: 2026-06-28  
기준 버전: `0.1.0+1`  
대상: Flutter 앱 MVP 및 Railway API 연동

## 01. 제품 개요

LoveType Tarot은 사용자의 생년월일 기반 소울카드, MBTI, 성별, 선택 키워드, 타로 카드 3장을 조합해 AI 스토리형 리딩을 제공하는 모바일 앱이다.

## 02. 사용자

- 오늘의 감정 흐름을 짧게 확인하고 싶은 사용자
- 연애, 썸, 재회, 연락, 상대 마음 등의 키워드로 이야기를 보고 싶은 사용자
- 시각 접근성이 필요한 사용자
- 결과를 저장하거나 공유하고 싶은 사용자

## 03. 목표

- 온보딩에서 사용자 프로필과 소울카드를 생성한다.
- 오늘의 이야기와 연애 이야기의 리딩 플로우를 제공한다.
- 카드 선택 후 서버 AI 리딩 결과를 표시한다.
- 포인트/구독 상태와 프리미엄 덱 표시를 연결한다.
- 푸시/로컬 알림으로 재방문과 쿨타임 해제를 지원한다.
- 서버 쿨타임을 단일 기준으로 사용해 중복 리딩 정책을 일관되게 유지한다.

## 04. 주요 기능 범위

- Google 계정 기반 회원가입/로그인
- 닉네임, 성별, MBTI, 생년월일 입력
- 소울카드 번호 계산 및 이미지 표시
- 주제별 키워드 선택
- 7장 중 3장 카드 선택
- 과거/현재/미래 카드 확인
- AI 리딩 결과 생성
- 결과 TTS, 공유, 저장
- 포인트 조회, 충전, 사용
- 구독 여부 기반 웹툰 덱 표시
- 고대비 모드와 저시력용 덱
- FCM 토큰 등록 및 로컬 알림

## 05. 비범위

- 정식 스토어 릴리즈 서명 자동화
- 서버 운영 콘솔 UI 완성
- 모든 API의 end-to-end 통합 테스트
- 앱 번들 용량 최적화 완료
- 딥링크/복잡한 라우팅 정책

## 06. 성공 기준

- `flutter analyze` 통과
- `flutter test` 통과
- `flutter build apk --debug` 성공
- 메인 화면이 서버 쿨타임을 조회하고 잠금 상태를 표시
- 버튼 탭 시 서버 쿨타임을 재확인한 뒤 리딩 진입
- 결과 생성 후 서버 기준 쿨타임 알림 예약
- 핵심 문서가 Outline 위키로 게시되고 로컬 원본이 저장소에 남음

## 07. 리스크

- 서버 `tarot/cooltime` 응답 필드와 앱 파서 불일치
- PNG 덱 3종 포함으로 인한 APK/AAB 용량 증가
- 릴리즈 signing config가 아직 debug signing
- `baseUrl`이 코드 상수로 고정되어 staging/prod 분리가 미흡
- 포인트 차감과 리딩 생성 실패 재시도 시나리오의 추가 QA 필요
