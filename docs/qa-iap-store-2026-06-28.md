# LoveType Tarot 인앱결제 QA 체크리스트

작성일: 2026-06-28  
목표: 실제 스토어 상품 등록 직후 내부 테스트에서 확인할 항목

## 01. 준비

- Android release signing 준비
- Google Play 내부 테스트 트랙 업로드
- 테스트 계정 등록
- 스토어 상품 상태 활성화
- Railway `/payment/charge`, `/payment/balance`, `/payment/use` 정상 동작 확인

## 02. 상품 조회

앱 실행 후 결제 다이얼로그에서 다음을 확인한다.

- 스토어 사용 가능 환경에서 상품 구매 플로우가 열린다.
- 상품이 없으면 `스토어에서 상품을 찾지 못했습니다.` 메시지가 뜬다.
- 스토어 사용 불가 환경에서 서버 직접 충전이 발생하지 않는다.

## 03. 소모품 구매

대상 상품: `tarot_10p`

1. 연애 이야기 진입
2. 포인트 부족 상태 만들기
3. `1,000원 충전` 선택
4. 테스트 결제 완료
5. 서버 `/payment/charge` 호출 확인
6. `/payment/balance` 갱신 확인
7. 잔액이 10P 증가했는지 확인
8. 동일 구매 재처리 시 중복 증가가 없는지 확인

## 04. 구독 구매

대상 상품: `tarot_sub`

1. 결제 다이얼로그에서 월 구독 선택
2. 테스트 결제 완료
3. 서버 `/payment/charge` 호출 확인
4. `/payment/balance` 또는 구독 조회에서 `is_subscribed=true` 확인
5. 웹툰 덱 표시가 활성화되는지 확인
6. 복원 구매 시 중복 포인트 지급이 없는지 확인

## 05. 실패/취소

- 구매 취소 시 다이얼로그가 닫히거나 상태가 멈추지 않는지 확인
- 서버 검증 실패 시 포인트/구독이 반영되지 않는지 확인
- 네트워크 오류 후 재시도해도 중복 반영이 없는지 확인
- 스토어 상품 미등록 상태에서 사용자에게 명확한 오류가 표시되는지 확인

## 06. 로그 확인

확인할 로그 키워드:

- `[IapService] 상품 없음`
- `[IapService] buy error`
- `[IapService] charge API 실패`

릴리즈 빌드에서는 민감한 영수증/토큰 원문을 출력하지 않는다.
