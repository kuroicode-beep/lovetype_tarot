================================================================
02. 이 테스트의 전략적 역할
================================================================

LoveType 전체 흐름에서의 위치
- 테스트는 별도 콘텐츠가 아니라 본앱 유입 엔진이다
- 사용자는 먼저 재미있는 테스트로 들어온다
- 결과에서 "꽤 잘 읽어준다"는 신뢰를 형성한다
- 이후 LoveType 앱의 타로 이야기로 자연스럽게 이어진다

핵심 전략
- 오늘의 타로/이야기 앱이 탄탄하다는 인상을 주기 전에
  먼저 연애유형 테스트로 대중적 관심을 끌어온다
- MBTI 결과를 강도 표현까지 확장해 차별성을 만든다
- 결과 화면에서 "더 많은 이야기가 있습니다" 흐름으로
  LoveType 본앱 CTA를 붙인다

================================================================
03. 강점 / 약점 / 극복 포인트
================================================================

강점
- 누구나 관심이 있다
- 진입장벽이 낮다
- 결과 공유가 쉽다
- SNS 확산 가능성이 높다

약점
- 누구나 비슷한 것을 만들 수 있다
- 흔한 성격유형 테스트처럼 보이기 쉽다
- 차별점이 약하면 금방 묻힌다

극복 포인트 1
성격유형 테스트로부터 자연스러운 유입 유도

설명
- 타로앱을 정면으로 권하는 대신
  MBTI 연애유형 테스트를 먼저 제공한다
- 테스트 자체가 유입 콘텐츠이자 공유 콘텐츠 역할을 한다

극복 포인트 2
단순 카드뽑기에서 벗어나
"나의 이야기가 있는 앱" 이미지 만들기

설명
- 결과 해설의 깊이
- LoveType 본앱 연결
- 추후 소울카드/히스토리와 결합
이 구조로 "내 앱" 같은 감각을 만든다

극복 포인트 3
데이터 정교화 + LLM 품질 향상으로 해설 차별화

설명
- 단순 MBTI 결과문이 아니라
  축별 강도 표현과 해설 밀도로 차별화
- 테스트 결과가 LoveType 본앱의 개인화 이야기와 연결되게 설계

04. 핵심 차별화 포인트
================================================================

4-1. 일반 MBTI보다 세밀한 강도 표현

일반 MBTI
- E 또는 I
- S 또는 N
- T 또는 F
- J 또는 P

LoveType 방식
- 5문항 중 몇 개를 선택했는지 비율까지 반영
- 단순 16유형을 넘어 강도 표현 포함

예시
- E(강)
- E(중)
- E(약)
- I(약)
- I(중)
- I(강)

핵심 가치
- 같은 ENFP라도 결이 다르게 느껴지게 한다
- 결과에 개인화 체감을 준다
- "나를 좀 더 잘 읽어준다"는 인상을 만든다

4-2. 결과 시각화
- 4축 강도 그래프 제공
- EI / SN / TF / JP 각각의 기울기 표시
- 텍스트 수치와 그래프 병기
- 일반 모드와 고대비 모드 모두 읽기 쉽게 구성

4-3. 결과 해설의 밀도
- 한 줄 총평
- 연애 스타일 요약
- 관계에서의 강점
- 조심할 포인트
- 잘 맞는 흐름 또는 관계 성향
- 공유용 짧은 결과문

12. URL 파라미터 접근성 규칙
================================================================

목적
- 링크만으로도 처음부터 고대비 모드 진입 가능하게 한다
- 접근성 사용자에게 별도 바로가기 제공 가능하게 한다

규칙
- mode=hc : 고대비 모드로 시작
- mode가 없으면 기본 일반 모드
- localStorage에 저장된 mode가 있으면 그 값을 사용
- 단, URL 파라미터가 있으면 localStorage보다 우선

예시 URL
- /index.html
- /index.html?mode=hc

우선순위
1. URL 파라미터
2. localStorage 저장값
3. 기본 일반 모드

의도
- 접근성 바로가기 링크 지원
- 결과 공유 시 현재 모드 유지 확장 가능
- 저시력 사용자 반복 편의 향상

================================================================

- 결과의 무게감 유지
- 반복 새로고침식 소비 방지

제한 구조
1차 프론트 제한

2차 서버/API 제한
- user_id 또는 guest_token
- IP 또는 간단한 보조 식별
- KST 기준 날짜 단위 제한

권장 문구
- 오늘의 테스트는 이미 완료했어요
- 다시 테스트는 1일 1회만 가능합니다

22. 이 테스트의 최종 역할
================================================================

LoveType 연애유형 테스트는
그 자체로 가볍게 퍼질 수 있는 콘텐츠이면서,
동시에 LoveType 본앱으로 이어지는
가장 중요한 첫 입구다.

이 테스트의 목표는
단순한 유형 놀이가 아니라

- 흥미를 만들고
- 신뢰를 만들고
- 더 깊은 이야기로 이어지게 만드는 것

이다.

================================================================
23. 한 줄 전략 정리
================================================================

LoveType 연애유형 테스트는
가볍게 시작해서 결과에서 신뢰를 만들고,
그 신뢰를 LoveType의 더 깊은 이야기로 이어주는
원페이지형 유입 엔진이다.
